# Rota DB Models — Design

## Overview

A Prisma/Postgres data model for managing coworker desk usage at coworking locations. Coworkers follow a weekly rota plan that generates concrete future days. Days can be swapped between users peer-to-peer without admin approval.

Scope: **database models and queries only**. No front-end components or business logic.

---

## Core Entities

### Location

A physical coworking location. Contains desks. Users are assigned to exactly one location.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `name` | String | Display name |
| `address` | String | Physical address |

### Desk

An individual desk at a location. Used for capacity tracking — rota days are not assigned to a specific desk (hot-desking).

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `name` | String | e.g. "Desk 4A" |
| `locationId` | String | FK → Location |

### User

Synced from Clerk. Assigned to a single location.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `clerkId` | String | Unique, from Clerk |
| `name` | String | |
| `email` | String | Unique |
| `locationId` | String | FK → Location |

## Rota Entities

### RotaPlan

The recurring weekly pattern for a user. Multiple plans can exist per user (each with a different `effectiveFrom`). The active plan is the most recent one where `effectiveFrom <= today`.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `userId` | String | FK → User |
| `effectiveFrom` | DateTime | Plan start date |
| `effectiveUntilSoft` | DateTime | 1 month after `effectiveFrom` — soft horizon for rota day generation |
| `effectiveUntilHard` | DateTime | 1 year after `effectiveFrom` — absolute limit for rota day generation |

### RotaPlanDay

Each row represents one day of the week included in a plan.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `planId` | String | FK → RotaPlan |
| `dayOfWeek` | Int | 0 = Sunday … 6 = Saturday |

Unique constraint: `(planId, dayOfWeek)`.

### RotaDay

A concrete calendar date allocated to a user. Generated by application logic from the user's active `RotaPlan`. Ownership (`userId`) can change via a swap.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `userId` | String | FK → User (current owner) |
| `date` | DateTime | The calendar date (time normalised to midnight UTC) |
| `status` | Enum | `PAID`, `CONTRACTED`, or `null` (unset) |

Unique constraint: `(userId, date)`.

**RotaDayStatus enum:** `PAID`, `CONTRACTED`

---

## Swap Entities

Swaps are peer-to-peer. No admin approval required. Three scenarios are supported:

1. **Full swap** — User A requests a date and offers alternatives; User B provides the wanted date and takes one of the offered days.
2. **One-sided give** — User B accepts User A's request and provides their day without taking any of User A's offered days.
3. **Standalone provide** — User B proactively marks a day as free (no matching request exists yet); any user can claim it.

### SwapRequest

User A's request to come in on a specific date, with a list of days they are willing to give up.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `requesterId` | String | FK → User (User A) |
| `wantedDate` | DateTime | The date User A wants to come in |
| `status` | Enum | `OPEN`, `FULFILLED`, `CANCELLED` |
| `createdAt` | DateTime | Auto |

**SwapRequestStatus enum:** `OPEN`, `FULFILLED`, `CANCELLED`

### SwapOffer

Each row is one of User A's offered alternatives — a `RotaDay` they are willing to give up as part of the swap.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `swapRequestId` | String | FK → SwapRequest |
| `rotaDayId` | String | FK → RotaDay (must belong to requester) |

### SwapAcceptance

User B's acceptance of a swap request. Records which day User B is providing and (optionally) which of User A's offered days they are taking.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `swapRequestId` | String | Unique FK → SwapRequest |
| `acceptorId` | String | FK → User (User B) |
| `providedRotaDayId` | String | FK → RotaDay (User B's day, given to User A) |
| `acceptedOfferId` | String? | Nullable FK → SwapOffer (the day User B takes from User A; null = one-sided give) |
| `resolvedAt` | DateTime | When the swap was completed |

**On resolution (application logic):**
- `providedRotaDayId.userId` → updated to User A (`requesterId`)
- If `acceptedOfferId` is set: that `SwapOffer.rotaDayId.userId` → updated to User B (`acceptorId`)
- `SwapRequest.status` → `FULFILLED`

### StandaloneOffer

User B proactively marks a day as free with no matching request. Any user at the same location can claim it.

| Field | Type | Notes |
|---|---|---|
| `id` | String (cuid) | PK |
| `offererId` | String | FK → User (User B) |
| `rotaDayId` | String | Unique FK → RotaDay |
| `status` | Enum | `OPEN`, `CLAIMED`, `CANCELLED` |
| `claimedById` | String? | Nullable FK → User (whoever claims it) |
| `claimedAt` | DateTime? | When claimed |

**StandaloneOfferStatus enum:** `OPEN`, `CLAIMED`, `CANCELLED`

**On claim (application logic):**
- `rotaDayId.userId` → updated to `claimedById`
- `StandaloneOffer.status` → `CLAIMED`

## Relationships Summary

```
Location
  ├── Desk[]
  └── User[]
        ├── RotaPlan[]
        │     └── RotaPlanDay[]
        ├── RotaDay[]
        │     └── StandaloneOffer? (one)
        ├── SwapRequest[] (as requester)
        │     ├── SwapOffer[]
        │     └── SwapAcceptance (one)
        ├── SwapAcceptance[] (as acceptor)
        └── StandaloneOffer[] (as offerer or claimer)
```

## Queries (Prisma)

### Get a user's rota days (upcoming, ordered)

```ts
prisma.rotaDay.findMany({
  where: {
    userId,
    date: { gte: new Date() },
  },
  orderBy: { date: 'asc' },
})
```

### Get the active rota plan for a user

```ts
prisma.rotaPlan.findFirst({
  where: {
    userId,
    effectiveFrom: { lte: new Date() },
  },
  orderBy: { effectiveFrom: 'desc' },
  include: { days: true },
})
```

### Get all open swap requests at a user's location (for browsing)

```ts
prisma.swapRequest.findMany({
  where: {
    status: 'OPEN',
    requester: { locationId },
  },
  include: {
    requester: true,
    offers: { include: { rotaDay: true } },
  },
})
```

### Create a swap request with offered days

```ts
prisma.swapRequest.create({
  data: {
    requesterId,
    wantedDate,
    status: 'OPEN',
    offers: {
      create: offeredRotaDayIds.map((rotaDayId) => ({ rotaDayId })),
    },
  },
})
```

### Accept a swap (full or one-sided) — wrapped in a transaction

```ts
prisma.$transaction(async (tx) => {
  // 1. Record acceptance
  await tx.swapAcceptance.create({
    data: {
      swapRequestId,
      acceptorId,
      providedRotaDayId,
      acceptedOfferId: acceptedOfferId ?? null,
      resolvedAt: new Date(),
    },
  })

  // 2. Transfer provided day to requester
  await tx.rotaDay.update({
    where: { id: providedRotaDayId },
    data: { userId: requesterId },
  })

  // 3. Transfer accepted offered day to acceptor (full swap only)
  if (acceptedOfferId) {
    const offer = await tx.swapOffer.findUniqueOrThrow({
      where: { id: acceptedOfferId },
    })
    await tx.rotaDay.update({
      where: { id: offer.rotaDayId },
      data: { userId: acceptorId },
    })
  }

  // 4. Close the request
  await tx.swapRequest.update({
    where: { id: swapRequestId },
    data: { status: 'FULFILLED' },
  })
})
```

### Get all open standalone offers at a location (scenario 3)

```ts
prisma.standaloneOffer.findMany({
  where: {
    status: 'OPEN',
    offerer: { locationId },
    rotaDay: { date: { gte: new Date() } },
  },
  include: {
    offerer: true,
    rotaDay: true,
  },
  orderBy: { rotaDay: { date: 'asc' } },
})
```

### Claim a standalone offer

```ts
prisma.$transaction(async (tx) => {
  await tx.standaloneOffer.update({
    where: { id: standaloneOfferId },
    data: {
      status: 'CLAIMED',
      claimedById,
      claimedAt: new Date(),
    },
  })

  await tx.rotaDay.update({
    where: { id: rotaDayId },
    data: { userId: claimedById },
  })
})
```
